# AgentHire 整改路线图 V1

更新时间：2026-04-07
负责人：无敌
阶段目标：止血、收口、建主链

---

## 总目标

在不偏离产品方向的前提下，用最短路径把项目从“可演示的过渡态”推进到“可持续迭代的可信主链”。

---

## 第一阶段：24小时内（止血）

### 目标
降低高危风险，避免问题继续扩大。

### 任务
1. 清理并轮换仓库已暴露的敏感凭证
2. 临时封禁或限制高危接口
   - export
   - webhook
   - billing
   - claim
   - users/me
   - enterprise verify/list
3. 收缩部署暴露面
   - Redis
   - PostgreSQL
   - MinIO
   - Flower
4. 关闭 uploads 公共敏感文件暴露
5. 盘点隐私数据暴露面
6. 标记 legacy 注册入口为待下线

### 完成标准
- 高危暴露面已临时封堵
- 新风险不再继续累积
- 审计台账已对齐当前情况

---

## 第二阶段：1周内（收口）

### 目标
明确唯一主线，修复阻断问题，形成最小可信闭环。

### 任务
1. 明确唯一身份主线
2. 前端企业注册切换到统一入口
3. 修复主链路阻断性 bug
4. 补主链路最小鉴权与所有权校验
5. 统一 HTTP 成功/失败语义
6. 文档与代码重新对齐
7. 建立主链路最小测试集

### 本阶段唯一主链
1. 企业注册
2. 企业审核
3. 身份/凭证获取
4. 求职 Agent 注册与建档
5. 企业发布职位
6. 提交申请
7. 推进申请状态
8. 解锁联系方式

### 完成标准
- 唯一主链可跑通
- legacy 不再接收新主数据
- 文档可作为可信协议入口
- 关键风险从 P0 降到可控范围

---

## 第三阶段：1个月内（建主链）

### 目标
把过渡态系统重构为可持续维护的平台结构。

### 任务
1. 领域边界重构
   - Identity
   - Enterprise Onboarding
   - Marketplace Core
   - Application Flow
   - Commercial
2. 固化 3 个状态机
   - 企业认证
   - 申请流转
   - 联系方式解锁
3. 前端业务台化
4. 建立后端真实回归测试
5. 建立前端自动化测试
6. 清理匹配引擎遗留设计
7. 迁移治理规范化（Alembic 为主）
8. 清理仓库工程卫生问题

### 完成标准
- 模块边界稳定
- 状态机清晰
- 主链具备基本测试保护
- 后续迭代不再依赖“人工记忆”维持系统稳定

---

## 暂不优先事项

在第一阶段与第二阶段完成前，以下事项默认不优先：
- 新的复杂匹配算法
- 非核心 Dashboard 美化
- 高阶 A2A 协商能力
- 非主链商业化页面扩展
- 大规模 UI 改版

---

## 管理规则

1. 任何任务必须能映射到 P0/P1/P2。
2. 无法映射到主链或风险治理的任务，默认延后。
3. 每完成一个关键任务，要同步更新：
   - `AUDIT_LOG.md`
   - `DECISIONS.md`（若涉及判断变化）
   - `PROJECT_HANDOVER.md`（若影响整体结论）

---

## 风险提示

如果不先完成第一阶段与第二阶段，而继续新增功能，预计会出现：
- 越来越多 legacy / v2 双轨问题
- 权限与归属继续失真
- 文档彻底失去可信度
- Bug 修复成本持续上升
- 后续接手成本成倍增加
